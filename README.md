# ☕ Nada Coffee Frontend Project

나다커피(Nada Coffee)의 신선한 원두와 음료를 만나볼 수 있는 프리미엄 커머스 플랫폼입니다.

---

## 🚨 백엔드 개발자 요청 사항 (Urgent Requests)

원활한 기능 완성을 위해 다음 API 및 DB 설정 변경을 요청드립니다.

### 1. 포인트 시스템 (Point System)
- **구매확정 자동 적립**: 주문 상태가 `PURCHASE_COMPLETED`로 변경될 때, 백엔드 내부 로직(트랜잭션)을 통해 결제 금액의 1%가 자동 적립되도록 구현 부탁드립니다. (현재 프론트에서 수동 호출 중이나 보안상 백엔드 처리가 권장됨)

### 2. 리뷰 시스템 (Review System)
- **리뷰 이미지 다중 업로드**: 현재 리뷰 등록 시 `imageUrls` 배열을 지원하는지 확인 부탁드립니다.

## 📅 작업 현황 (2026-02-09)

### ✅ 오늘 완료한 작업 (Today's Achievements)

#### 1. 주문 및 결제 프로세스 고도화 💳
- **결제 모달 개선**: 토스페이먼츠 결제 위젯 모달의 레이아웃을 구조화하여, 어떤 단계에서도 **[취소하기]** 버튼이 항상 하단에 고정 노출되도록 UX 개선.
- **주문 상태 명세 반영**: 백엔드의 새로운 주문 상태(`PAYMENT_COMPLETED`, `PREPARING`, `SHIPPING`, `DELIVERED`, `PURCHASE_COMPLETED`, `CANCELLED`, `RETURNED`)를 시스템 전반에 적용.
- **한글/영어 상태값 호환**: 서버에서 영어 전문 용어 또는 한글 상태명을 보내더라도 프론트엔드에서 정확히 인식하여 배지와 라벨을 표시하도록 로직 강화.

#### 2. 관리자(Admin) 기능 최적화 👑
- **주문 상태 일괄 관리**: 관리자 주문 목록의 ACTIONS 컬럼을 `select` 박스로 교체하여, 마우스 호버 없이도 즉시 상태를 변경할 수 있도록 개선.
- **대시보드 통계 복구**: 오늘 매출, 주문 건수, 신규 회원, 처리 대기 건수 등 핵심 지표 계산 로직을 복구하고 실시간 차트에 반영.
- **재고 알림 디자인 통일**: Stock Alerts 섹션의 디자인을 대시보드의 다른 카드들과 일관성 있게(흰색 배경 + 그림자) 리뉴얼.
- **보안 강화 (Route Guard)**: 관리자 페이지 접근 시 `role`을 엄격히 체크하여 일반 유저의 접근을 원천 차단하고 메인으로 리다이렉트 처리.
- **로그아웃 기능**: 관리자 헤더 우측에 직관적인 로그아웃 버튼 추가.

#### 3. 마이페이지(My Page) 및 포인트 시스템 💰
- **구매확정 및 적립**: 배송 완료된 주문에 대해 사용자가 직접 **[구매확정]**을 할 수 있는 기능을 추가하고, 확정 시 결제액의 **1% 포인트 적립** 안내 로직 구현.
- **주문 취소 기능 확대**: 상세 보기 없이도 목록에서 즉시 주문 취소가 가능하도록 버튼 배치 (결제대기~배송준비 단계).
- **포인트 내역 페이지네이션**: 포인트 이용 내역이 많을 경우를 대비해 10개 단위의 페이지네이션 기능을 추가하여 데이터 조회 효율성 증대.

#### 4. 리뷰(Review) 시스템 개선 ✍️
- **데이터 전송 최적화**: 리뷰 등록 시 발생하는 400 에러를 해결하기 위해 페이로드 구조를 정밀 조정(이미지 없을 시 필드 제외, 타입 강제 변환 등).
- **API 경로 정상화**: 리뷰 등록/수정/삭제 시 발생하는 404 에러를 해결하기 위해 중복된 `/api` 경로 제거.
- **이동 경로 수정**: 리뷰 등록 성공 후 잘못된 경로(`/menu/...`)로 이동하여 발생하는 404 에러를 실제 상품 상세 페이지인 **`/products/:id#reviews`**로 수정.

#### 5. 코드 품질 및 아키텍처 개선 🛠️
- **관심사 분리 (SoC)**: 비대해진 `MyPage.tsx`를 `useMyPage` 커스텀 훅과 기능별 컴포넌트로 분리하여 가독성 및 유지보수성 극대화.
- **클래스명 관리 최적화**: 프로젝트 전체의 Tailwind CSS 클래스명을 `twMerge([...])` 배열 형태로 리팩토링하여 속성별 그룹화 및 스타일 충돌 방지.

## 🛠 Tech Stack
- **Frontend**: React 19, TypeScript, Vite
- **Styling**: Tailwind CSS v4, Framer Motion, tailwind-merge
- **State Management**: Zustand, TanStack Query v5
- **Integration**: Toss Payments, Daum Postcode API
