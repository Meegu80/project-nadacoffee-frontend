# ☕ Nada Coffee Frontend Project

나다커피(Nada Coffee)의 신선한 원두와 음료를 만나볼 수 있는 프리미엄 커머스 플랫폼입니다.


## 📅 작업 현황 (2026-02-06)

### ✅ 오늘 완료한 작업 (Today's Achievements)

#### 1. 결제 시스템 최종 안정화 (Toss Payments) 💳
- **ID 매칭 로직 개선**: 토스페이먼츠의 `orderId` 제약(6자 이상)을 충족하기 위해 `ORDER_{id}_{timestamp}` 형식을 도입하고, 승인 시 서버에 문자열 그대로 전달하도록 최적화.
- **중복 호출 방지**: `useRef`를 활용하여 결제 승인 API가 단 한 번만 호출되도록 보장 (토스 400 에러 방지).
- **장바구니 클린업**: 결제 성공 시 서버 장바구니 비우기(`DELETE /api/cart`) 및 로컬 스토어 초기화 로직 연동.

#### 2. 메인 페이지(Home) 대대적 개편 🏠
- **히어로 섹션**: 배경 영상을 최신 유투브 영상으로 교체하고, 브랜드 스토리/메뉴/매장찾기 바로가기 링크 연결.
- **매장 찾기 섹션(MainSection2)**: 실제 매장 정보(강남본점, 안산단원점 등)로 업데이트하고, 클릭 시 매장 찾기 페이지의 지도와 연동.
- **인기 메뉴 섹션(MainSection3)**: 하드코딩 데이터를 제거하고 **실제 상품 API** 연동. 최신 상품 4개를 큼직한 이미지(`h-[400px]`)와 함께 노출.
- **HOT 배지 도입**: 인기 메뉴에 12각형 별 모양의 강렬한 **HOT 배지**를 입체적으로 적용.

#### 3. 상품 목록 페이지 고도화 (Menu Page) ☕
- **페이지네이션 구현**: 한 페이지당 **18개**의 상품을 보여주는 클라이언트 사이드 페이지네이션 도입.
- **카테고리 필터링 정상화**: URL 경로와 카테고리를 완벽히 동기화하여 GNB/LNB 클릭 시 즉시 필터링되도록 개선.
- **재고 상태 표시**: 수량이 0일 때 **SOLD OUT**, 5개 이하일 때 **품절임박** 배지를 노출하여 구매 긴박감 조성.
- **서버 에러 우회**: 특정 카테고리 조회 시 발생하는 서버 500 에러를 방지하기 위해 100개씩 순차적으로 데이터를 가져와 병합하는 로직 적용.

#### 4. 마이페이지 및 관리자 기능 강화 👤
- **주문 내역 일괄 관리**: 체크박스를 통한 **다중 주문 취소(삭제)** 기능 추가.
- **주문 상세 모달 보완**: 데이터 로딩 안전 장치 추가 및 디버깅용 로그 시스템 구축.
- **관리자 대시보드**: 실제 주문 데이터를 기반으로 **오늘의 매출, 주간 매출 추이 차트**를 실시간 계산하여 반영.
- **회원 정보 수정**: 내 정보 수정 및 비밀번호 변경 Mutation 로직 완성.

#### 5. 공통 UI/UX 개선 ✨
- **브랜드 로고 통합**: 헤더, 푸터, 관리자 사이드바의 로고를 세련된 **텍스트 기반 로고**로 통일.
- **푸터 SNS 연동**: 인스타그램, 페이스북, 유투브 실제 링크 연결 및 레이아웃 조정.
- **웹 에디터 업로드**: 문의하기 에디터에 폰트 크기, 정렬, 색상 선택(스포이드) 등 고급 기능 추가.

---

## 🛠 Tech Stack
- **Frontend**: React 19, TypeScript, Vite
- **Editor**: Tiptap (Rich Text Editor)
- **Styling**: Tailwind CSS v4, Framer Motion
- **State Management**: Zustand, TanStack Query v5
- **Integration**: Toss Payments, Daum Postcode API
