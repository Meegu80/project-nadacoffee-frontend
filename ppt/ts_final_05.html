TROUBLESHOOTING 05

Bestseller Logic: Problem Analysis
HIERARCHY

다중 계층 데이터 플래닝

Analysis

**전체 주문 -> 개별 주문 -> 상세 품목(Order Items)**으로 이어지는 3단계 배열을 하나씩 뒤져야 하는 구조적 어려움이 있었습니다. 수백 개의 주문 데이터를 실시간으로 전수 조사하여 상품 ID별로 재분류하는 과정에서 연산 복잡도가 크게 증가하는 문제가 발생했습니다.

Type: Logic Complexity

코드 위치: useBestSellers.ts
// 3중 중첩 데이터를 펼쳐야 함
orders.map(order => 
  order.items.map(item => ...))
MERGING

분산된 옵션 데이터 통합

Analysis

서버 데이터 상 **'아메리카노(HOT)'**와 **'아메리카노(ICE)'**는 서로 다른 줄(Row)로 오지만, 베스트셀러 순위에서는 이를 하나의 **'아메리카노'**라는 상품 ID로 합성해야 했습니다. 옵션명에 관계없이 "순수 상품 ID"로만 묶어내는 로직이 핵심이었습니다.

Type: Data Inconsistency

기대 결과: 단일 ID 집계
// 옵션 무시, 단일 ID 집계
{ productId: 101, totalQty: 50 }
FETCH LIMIT

대량 조회(Limit 100) 누락

Analysis

집계를 위해 100개 이상의 상품을 한꺼번에 가져오려 했으나, 서버 성능 보안상 한 페이지당 50개로 제한을 걸어두어 요청이 무시되거나 데이터가 누락되는 현상이 발생했습니다.

Status: 400 | Console: Bad Request

현상: 데이터 잘림
// Request: limit=100
// Response: 50 items (Silent)
