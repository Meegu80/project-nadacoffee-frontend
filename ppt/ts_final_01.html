MULTIPART ERROR

파일 업로드 서버 에러 (500)

Analysis

이미지 업로드 시 파일 이름에 한글이나 특수문자가 포함되면 서버가 해석하지 못해 업로드 자체가 실패하고 서버가 멈추는 현상이 있었습니다.

Status: 500 | Console: PayloadTooLargeError

Resolution

파일을 보낼 때 원본 이름 대신 upload_타임스탬프 형식으로 이름을 강제 변경하여 서버와의 문자 인코딩 충돌을 원천 차단했습니다.

Stack: Axios, HTML
코드 위치: src/api/upload.api.ts (L16)
// 안전한 파일 이름으로 강제 변환
const extension = file.name.split('.').pop();
const safeName = `upload_${Date.now()}.${extension}`;
formData.append("file", file, safeName);
GCS 403 ERROR

이미지 로딩 권한 거부 (403)

Analysis

구글 클라우드 저장소(GCS)에 이미지는 잘 올라갔지만, 외부에서 이 이미지를 볼 수 있는 '읽기 권한'이 설정되지 않아 엑스박스가 뜨는 문제였습니다.

Status: 403 | Console: CORS Missing Allow Origin

Resolution

GCS 버킷 설정에서 모든 사용자(`allUsers`)에게 'Storage Object Viewer' 권한을 부여하고, CORS 정책을 통해 특정 도메인만 접근 가능하게 잠금 해제했습니다.

Stack: Google Cloud Storage
코드 위치: Cloud Console (CORS Policy)
// JSON 정책 적용 내용
[{"origin": ["*"], 
  "method": ["GET"],
  "maxAgeSeconds": 3600}]
