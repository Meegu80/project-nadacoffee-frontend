Product & Admin Page Troubleshooting

상품 목록 및 관리자의 데이터 핸들링 이슈

QUERY SYNC MENU LIST
무한 스크롤 데이터 중복 및 누락 원인 (Cause)

React Query의 fetchNextPage 호출 시 페이지 번호와 데이터 매핑 로직 불일치로 동일 데이터 반복 노출.

해결책 (Solution)

getNextPageParam 로직을 고도화하여 서버의 totalPages와 현재 page를 정확히 비교하고 로딩 상태를 스켈레톤 UI로 분기 처리.

NULL REFERENCE PRODUCT DETAIL
상품 이미지 렌더링 런타임 에러 원인 (Cause)

데이터 로딩 전 이미지 객체 접근 및 잘못된 URL 체계(HTTP/HTTPS 혼용)로 인한 엑박 발생.

해결책 (Solution)

Optional Chaining(?.) 적용 및 공통 ImageFallback 컴포넌트를 구현하여 에러 시 기본 로고 이미지로 대체되도록 방어 코드 구축.

STATE DESYNC ADMIN DASHBOARD
주문 상태 변경 시 캐시 미갱신 원인 (Cause)

어드민에서 주문 상태 업데이트(Mutation) 성공 후 목록 데이터가 자동으로 새로고침되지 않아 수정 전 데이터 유지.

해결책 (Solution)

onSuccess 콜백에서 queryClient.invalidateQueries를 사용하여 관련 캐시를 즉시 무효화하고 최신 리스트를 강제 페칭하도록 구현.

OVERLOAD ADMIN SEARCH
검색어 입력 시 과도한 API 호출 원인 (Cause)

키워드 입력마다 즉시 API를 호출하여 서버 부하 및 프론트엔드 UI 버벅임(Lag) 발생.

해결책 (Solution)

Lodash의 debounce(500ms) 기술을 적용하여 입력을 멈춘 후 일정 시간이 지났을 때만 검색 요청이 가도록 최적화.
