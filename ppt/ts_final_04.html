State Management & Async Optimization
STALE STATE

Mutation 후 목록 미갱신 동기화 이슈

Analysis

주문 취소나 승인 등 데이터 변경(Mutation) 후에도 브라우저 캐시에 옛날 데이터가 남아있어, 화면에는 변경 전 상태가 계속 노출되는 사용자 혼란 발생.

Type: Sync Error | Console: Query Cache Stale

Resolution

서버 데이터가 바뀌는 성공 시점에 TanStack Query의 invalidateQueries를 호출하여, 관련 캐시를 즉시 파기하고 최신 데이터를 다시 서버에서 가져오도록 강제함.

Lib: TanStack Query
코드 위치: src/pages/(admin)/orders/AdminOrderList.tsx (L196)
// 성공 시 즉시 'orders' 키를 가진 캐시를 무효화
onSuccess: () => {
  queryClient.invalidateQueries({ queryKey: ["admin", "orders"] });
}
THROTTLING

과도한 API 중복 호출 및 레이스 컨디션

Analysis

검색창에 글자를 칠 때마다 API 요청이 가면서 서버 부하가 커지고, 응답 순서가 뒤바뀌어 입력한 글자와 다른 예전 결과가 화면에 나타나는 레이스 컨디션(Race Condition) 발생.

Type: Sync Error | Console: Excessive Network Requests

Resolution

사용자의 타이핑이 끝날 때까지 기다렸다가(500ms) 마지막 검색어만 서버에 보내도록 Debounce 기법을 적용하여 불필요한 요청을 90% 이상 제거함.

Lib: React (useEffect), Timer API
코드 위치: src/pages/menu/MenuPage.tsx (L43)
// 유저가 입력을 멈추고 0.5초 뒤에만 서버 요청 실행
useEffect(() => {
  const handler = setTimeout(() => { setDebouncedSearchQuery(query); }, 500);
  return () => clearTimeout(handler);
}, [query]);
