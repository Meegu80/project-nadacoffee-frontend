CLS FIX

에러 메시지에 의한 레이아웃 시프트

Analysis

입력값이 틀렸을 때 에러 메시지가 '갑자기' 나타나면서 폼 전체가 아래로 뚝 떨어지는 현상입니다. 이는 사용자 시각을 방해하고 웹 성능 지수(CLS)를 떨어뜨립니다.

Type: Visual Bug | Tool: Lighthouse CLS

Resolution

메시지가 들어갈 자리를 미리 최소 높이(min-h-[14px])로 확보해둠으로써, 메시지가 나타나도 주변 UI가 움직이지 않게 고정했습니다.

Stack: Tailwind CSS
코드 위치: src/pages/(auth)/SignUp.tsx (L337)
// 자리를 미리 비워두어 흔들림 방지
<div className="min-h-[14px]">
  {errors.name && <p>{errors.name.message}</p>}
</div>
QUERY SYNC

무한 스크롤 중복 리스트 문제

Analysis

리스트 하단에 도달할 때 새 데이터를 불러오는데, 다음 페이지 번호를 잘못 계산하여 이미 보여준 상품들이 다시 중복해서 나타나는 버그였습니다.

Type: Logical | Console: Duplicate Key Warning

Resolution

데이터를 불러오는 핵심 파라미터(getNextPageParam)가 전체 페이지 한도 내에서만 정확히 +1씩 증가하도록 조건을 명확히 수정했습니다.

Lib: TanStack Query (v5)
코드 위치: src/pages/menu/ProductDetail.tsx (L94)
// 정확한 페이지 증감 로직
getNextPageParam: (last) => {
  return last.currentPage < last.totalPages 
    ? last.currentPage + 1 : undefined;
}
NULL SAFETY

데이터 로딩 중 흰 화면(Crash) 현상

Analysis

서버에서 데이터가 오기도 전에 빈 값(null)의 안쪽 속성(image 등)에 접근하려다 보니 자바스크립트가 에러를 내며 화면 자체가 하얗게 죽어버리는 현상입니다.

Status: Fatal | Console: Cannot read property of undefined

Resolution

Optional Chaining(?.)과 기본값 연산자(||)를 사용하여 데이터가 없을 때는 안전하게 비시각적 처리나 기본 이미지가 나오도록 보완했습니다.

Stack: TypeScript, JS ES2020
코드 위치: src/pages/menu/MenuPage.tsx (L232)
// 점진적 데이터 접근(Safe Reference)
const totalStock = product?.options?.reduce(
  (sum, opt) => sum + opt.stockQty, 0
) ?? 0;
