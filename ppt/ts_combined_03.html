Part 03: Admin & Optimization

어드민 운영 및 시스템 최적화

CACHE State Desync
주문 상태 업데이트 시 목록 미갱신 이슈
원인 (Cause)

어드민 대시보드에서 배송 상태 변경(Mutation) 성공 후, 브라우저에 캐싱된 이전 목록 데이터가 자동으로 Fresh 상태로 전환되지 않아 사용자가 수동으로 새로고침해야 하는 불편함 발생.

해결책 (Solution)

React Query의 invalidateQueries 기능을 mutation 성공 콜백(onSuccess)에 바인딩하여, 데이터 변경 즉시 관련 리스트 캐시를 무효화하고 배경에서 최신 데이터를 자동 로드하도록 구현.

LOAD Network Overload
대량 상품 검색 시 과도한 API 호출 이슈
원인 (Cause)

검색창에서 입력(Typing)이 발생할 때마다 즉각적인 API 요청을 전송하여, 짧은 시간 내 수십 번의 네트워크 오버헤드가 발생하고 전반적인 UI 버벅임(Lag) 및 서버 부하 증가.

해결책 (Solution)

Lodash 라이브러리의 Debounce(500ms) 기술을 적용하여 입력을 마친 일정 시간 후에만 최종 요청을 보내도록 최적화하고, 검색 로직의 안정성과 사용자 경험을 대폭 향상.
