Major Troubleshooting Cases

개발 과정의 시행착오와 해결책

500 ERROR API SERVER
이미지 업로드 내부 서버 오류 원인 (Cause)

업로드 폴더명 불일치 및 한글 파일명 인코딩 이슈로 인한 멀티파트 데이터 처리 실패.

해결책 (Solution)

허용된 폴더명(products)으로 동기화 및 업로드 전 파일명을 SafeFilename으로 자동 치환하는 로직 구현.

403 ERROR STORAGE
이미지 리소스 접근 권한 오류 원인 (Cause)

GCS(Google Cloud Storage) 버킷의 CORS 설정 및 익명 사용자 읽기 권한 미흡.

해결책 (Solution)

버킷의 공개 읽기(AllUsers) 권한 설정 및 도메인별 CORS 화이트리스트 최적화를 통한 리소스 연결 완료.

401 ERROR AUTH HEADER
API 인증 키(Client-Key) 누락 원인 (Cause)

Axios에서 Fetch API로 전환하는 과정에서 필수 헤더인 'x-client-key' 주입 누락.

해결책 (Solution)

공통 API 모듈 내 인터셉터(또는 래퍼) 기능을 강화하여 모든 비회원/회원 요청에 필수 키 자동 포함.

422 ERROR VALIDATION
데이터 유효성 검사 실패 원인 (Cause)

프론트엔드 Zod 스키마와 백엔드 DB 제약 조건(필드 길이, 형식 등)의 미세한 불일치.

해결책 (Solution)

서버 응답 에러 메시지를 React Hot Toast로 시각화하고, 스키마 정의를 API 스펙에 맞춰 재동기화.
