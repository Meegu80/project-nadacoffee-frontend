TROUBLESHOOTING 06

Bestseller Logic: Resolution Steps
STEP 1

3단계 데이터 평탄화

Implementation

중첩된 배열을 이중 루프(`forEach`)로 순회하며 상품 ID별 가방(`Map`)에 담았습니다. 1단계에서 날짜 필터링을 선행하여 불필요 연산을 사전에 차단했습니다.

Stack: JS Map, forEach
코드 위치: useBestSellers.ts
// 3단계 구조 관통 순회
orders.forEach(ord => {
  ord.items.forEach(it => { ... });
});
STEP 2

상품 ID 중심 통합 집계

Implementation

옵션이 달라도 상품 ID가 같다면 동일 상품으로 간주했습니다. `Map.get(id)`로 기존 데이터 유무를 확인하고 수량과 매출액만 누적 합산하여 병합했습니다.

Stack: Data Aggregation
코드 위치: useBestSellers.ts
// 상품 ID 기준 매출 합산
const cur = map.get(id) || { rev: 0 };
map.set(id, { rev: cur.rev + it.pay });
STEP 3

서버 제한 우회 호출

Implementation

클라이언트의 대량 요청을 서버 허용치인 50개 이하로 자동 조절했습니다. 부족한 양은 페이지 번호를 넘기며 여러 번 나누어 가져오는 '배치(Batch) 호출'로 해결했습니다.

Lib: TanStack Query
코드 위치: MenuPage.tsx (L56)
// Max 50 제한에 맞춰 분할 호출
