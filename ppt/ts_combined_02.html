Part 02: Data Management & UI

데이터 렌더링 안정화 사례

422 Unprocessable Entity
스키마 유효성 검사 불일치 (Zod vs DB)
원인 (Cause)

프론트엔드의 Zod 데이터 제약 조건과 백엔드 DB 설계 스펙 간의 미세한 불일치로 인한 회원가입/로그인 실패.

해결책 (Solution)

서버 응답 에러를 'React Hot Toast'로 가속 시각화하고, 최신 API 명세서에 맞춰 Zod 스키마를 동기화하여 검증 신뢰도 확보.

SYNC Infinite Scroll
상품 무한 스크롤 데이터 중복 및 누락 현상
원인 (Cause)

React Query의 fetchNextPage 호출 시 페이지 번호 계산 로직이 유동적인 서버 응답과 매핑되지 않아 동일 페이 데이터가 반복됨.

해결책 (Solution)

getNextPageParam 로직을 서버 pagination 스펙에 맞춰 재정의하고, 로딩 섹션에 스켈레톤 UI를 도입하여 자연스러운 인터랙션 구현.

NULL Reference Error
데이터 로딩 전 상품 이미지 참조 오류
원인 (Cause)

비동기 데이터가 완벽히 로드되기 전 이미지 객체 프로퍼티(src)에 접근하여 발생하는 화이트아웃(White-out) 현상.

해결책 (Solution)

Optional Chaining(?.) 문법 전면 도입 및 공통 ImageFallback 컴포넌트를 설계하여 누락되거나 비정상적인 이미지 리소스 예외 처리.
