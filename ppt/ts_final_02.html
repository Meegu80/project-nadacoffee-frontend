TROUBLESHOOTING 02

Authentication & Schema Validation
AUTH 401

API 인증 키 누락 에러 (401)

Analysis

모든 API 요청 시 '이 서비스의 정당한 요청자'임을 증명하는 x-client-key가 헤더에서 빠져있어 서버가 요청을 거절한 현상입니다.

Status: 401 | Console: Unauthorized Access

Resolution

Axios 인스턴스 설정 시 공통 헤더 영역에 환경 변수에서 가져온 인증 키를 기본값으로 박아넣어 수동 작업 없이 누락을 방지했습니다.

Lib: Axios, Vite Env
코드 위치: src/api/axios.ts (L9)
// 공통 헤더에 인증 키 영구 주입
const api = axios.create({
  headers: {
    'x-client-key': import.meta.env.VITE_API_KEY
  }
});
VALID 422

입력 값 유효성 검사 실패 (422)

Analysis

회원가입 폼에서 유저가 입력한 형식이 서버가 요구하는 엄격한 규칙(예: 비밀번호 특수문자 포함)과 맞지 않아 발생한 데이터 거절 문제입니다.

Status: 422 | Console: ZodValidationError

Resolution

Zod 라이브러리를 통해 서버와 동일한 정규식 검증 규칙을 프론트엔드에도 적용하여, 서버에 가기 전 미리 오류를 잡아내고 경고를 띄웠습니다.

Lib: Zod, React Hook Form
코드 위치: src/pages/(auth)/SignUp.tsx (L11)
// 서버 규격과 동일한 스키마 정의
const signUpSchema = z.object({
  password: z.string().min(8).regex(/^(?=.*[a-zA-Z])...$/)
});
