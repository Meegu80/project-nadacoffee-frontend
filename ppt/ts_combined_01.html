Part 01: Infrastructure & API

서버 및 인프라 장애 극복 사례

500 Internal Server Error
이미지 업로드 멀티파트 데이터 처리 실패
원인 (Cause)

서버 미배포 폴더명('inquiries') 호출 및 비정상적인 파일명(한글/특수문자) 인코딩 이슈로 인한 서버 런타임 중단.

해결책 (Solution)

허용된 서버 경로('products') 동기화 및 업로드 전 파일명을 UUID 기반 SafeFilename으로 치환하는 치밀한 전처리 프로세스 구축.

403 Forbidden
공공 스토리지 리소스 접근 및 권한 오류
원인 (Cause)

Google Cloud Storage 버킷의 CORS 보안 정책과 외부 익명 사용자의 읽기(Read) 권한 미설정으로 인한 리소스 차단.

해결책 (Solution)

GCS 버킷의 권한을 'allUsers-Viewer'로 최적화하고, 운영 도메인을 화이트리스트에 추가하여 안전한 리소스 연결 경로 확보.

401 Unauthorized
API 인증 필수 키(Client-Key) 누락
원인 (Cause)

비동기 라이브러리 교체(Axios → Fetch) 과정에서 기존 공통 헤더로 주입되던 'x-client-key' 값이 일부 요청에서 손실됨.

해결책 (Solution)

Fetch 래퍼(Wrapper) 함수를 제작하여 모든 프로젝트 요청에 공통 헤더가 자동 주입되도록 중앙 집중화된 API 통신부 리팩토링.
